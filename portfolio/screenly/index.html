Screenly

Digital signage made easy.

Worked with a multinational team of engineers to create both a paid and an open source version of Screenly, a distributed digital signage solution that runs on Raspberry Pi hardware.

Screenly Pro consists of a centralized, multi-tenant server cluster which periodically syncs with a thin-client version of Screenly OSE running on clients' Raspberry Pis.

Skills Used/Developed on this Project

On this project we used Python with the Bottle framework backed by MongoDB. Our client-side styles were written in LESS and compiled to CSS.

While on this project I learned CoffeeScript and Backbone.js. As one of my first major JavaScript-heavy projects, I rapidly self-educated myself with all JavaScript resources that I could find. It was part of my Annual Goals for 2013 to "build exceptional interactive experiences with JavaScript" and this project lent itself very well to that aspiration.

I also gained experience with lower-level hardware concerns as we built Raspian images of Linux pre-loaded with our Python code to run on a Raspberry Pi. While most of my time was spent with the front-end scripting, I have my hands in some of the Python and video playing components of the project, deploying code directly to a Raspberry Pi.
